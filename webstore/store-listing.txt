Chrome Web Store 掲載情報（コピペ用）
============================================================

【ストアの掲載情報】タブ
------------------------------------------------------------

■ 名前（30文字以内）
WEB制限解除サポート

■ 概要（132文字以内）
右クリック・テキスト選択の制限を自動解除。右クリックメニューから強制ペースト／強制コピー。SharePoint 等のセッション維持もワンタッチ。

■ 説明（16,000文字以内）
Webページのコピー・ペースト・右クリック・テキスト選択などの制限を解除するChrome拡張機能です。
右クリック禁止・コピー禁止・選択禁止などの制限がかかったページでも、ワンタッチで制限を解除できます。

■ 主な機能

・サイレント自動解除（トグル ON で常時動作）
  右クリック制限・テキスト選択制限を自動で解除します。
  Excel Online / Google Docs / Notion / Figma 等のカスタム右クリックメニューを提供するサイトは自動で判別し、サイト側のメニューを尊重します。

・右クリックメニューから実行
  ▸ 強制ペースト: ペーストが無効化された入力欄で「📋 強制ペースト」で貼り付け
  ▸ 強制コピー: コピーが無効化されたページで「✂️ 強制コピー」でクリップボードにコピー

・セッション維持（オプトイン・デフォルトOFF）
  SharePoint / Box 等でアイドル判定によるセッションタイムアウトを緩和します。
  全サイトで合成 mousemove を定期発射し、JS 側のアイドル検知をリセット。
  さらに SharePoint（*.sharepoint.{com,cn,de,us}）では同一オリジンへの軽量 GET も発行してサーバー側セッションを延長します。
  ※ 合成イベントのみではサーバー側セッションを維持できないサイト（Box 等）もあります。

・カスタム右クリックを許可するサイト（ユーザー追加）
  Office / Google Docs 等の主要 SaaS は組み込みで許可済みですが、独自のカスタム右クリックメニューを提供するサイトを追加で登録できます。ポップアップの「🖱️ カスタム右クリックを許可するサイト」セクションで 1 行 1 ドメインを入力してください。

■ 使い方

1. ツールバーのアイコンをクリックしてポップアップを開く
2. 「⚡ 制限解除」トグルを ON にする（即時適用）
3. 必要に応じて「🔄 セッション維持」トグルを ON にし、ポーリング間隔をスライダーで調整（1〜15 分）
4. 入力欄やテキスト選択中に右クリックして「強制ペースト／強制コピー」を実行
5. 必要に応じて「🖱️ カスタム右クリックを許可するサイト」にドメインを追加

設定は自動保存され、次回起動時も同じ状態が復元されます。

■ プライバシー

・個人情報・ユーザーデータの収集は一切行いません
・設定（4 キー: enabled / keepAliveEnabled / keepAliveIntervalMs / contextMenuAllowDomains）はお使いの端末内にのみ保存されます
・第三者の外部サーバーへの通信は一切行いません（セッション維持 ON 時、SharePoint 等ではアクセス中のサイト自身への同一オリジン GET が発生することがあります）

【プライバシーへの取り組み】タブ
------------------------------------------------------------

■ このアイテムに単一の目的はありますか？
はい

■ 単一の目的の説明
Webページの右クリック・テキスト選択の制限を解除し、強制ペースト／強制コピーを可能にする

■ データの使用に関する開示

  ・個人を特定できる情報を収集しますか？
    いいえ

  ・ユーザーのデータをどのように扱いますか？
    - ユーザーデータの販売: しない
    - ユーザーデータの本来の目的以外への使用や転送: しない
    - 信用力の判定や貸付目的での使用や転送: しない

■ 権限の正当性

  activeTab:
  ポップアップでトグル変更した際、現在のタブの content script に即時通知するために使用します。バックグラウンドでのデータアクセスは行いません。

  scripting:
  メインワールド（ページ側 JS コンテキスト）でインラインハンドラを除去するためのスクリプトを注入する目的で使用します。CSP の影響を回避するための最小限の使用です。

  storage:
  拡張機能の設定（enabled / keepAliveEnabled / keepAliveIntervalMs / contextMenuAllowDomains）をローカルに保存し、次回起動時に復元するために使用します。データは端末内にのみ保存され、外部に送信されることはありません。

  contextMenus:
  ユーザーがページ上で右クリックしたときに「強制ペースト」「強制コピー」の項目を表示するために使用します。

  clipboardRead:
  「強制ペースト」実行時にクリップボードの内容を読み取るために使用します。

  clipboardWrite:
  「強制コピー」実行時に選択テキストをクリップボードへ書き込むために使用します。

  offscreen:
  HTTP ページ上の content script では secure context 制限で navigator.clipboard が reject されるため、extension context の offscreen document 経由でクリップボード操作を実行するために使用します。

■ リモートコードを使用していますか？
いいえ
（すべてのコードは拡張機能パッケージ内に含まれており、外部からのコード読み込みは一切行いません。）
